# Gas Can 0.1.6 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the CLI version flags as Gas Can 0.1.6 through the existing signed macOS release pipeline.

**Architecture:** Keep the repository-prescribed version bump as one reviewable commit on PR #17, then run the complete verification suite before changing remote state. After a squash merge, anchor the exact merge commit with a verified signed tag and hand all packaging, notarization, GitHub publication, and Homebrew tap work to `packaging/macos/release.sh`.

**Tech Stack:** Rust/Cargo workspace, Bash release contracts and driver, Git/GitHub CLI, Apple codesigning and notarization, Homebrew cask tooling

## Global Constraints

- Release version is exactly `0.1.6`; tag is exactly `v0.1.6`.
- The version bump changes exactly the six workspace manifests, root `Cargo.lock`, `README.md`, and `docs/release/macos-checklist.md`.
- Leave `scripts/Cargo.lock` and `tests/release/release-script-contract.sh` unchanged.
- Never create the bump directly on `main`; add it to `feat/cli-version` and PR #17.
- The tag must be annotated, signed, verifiable, and target the exact squash commit on `main`; never use an unsigned fallback or move an existing tag.
- Run the release driver's non-mutating `--check` successfully before its live mode.
- Never replace a failed release-driver stage with ad hoc publication, deletion, or a force-push.
- Preserve unrelated changes in the main checkout and never commit `.superpowers/sdd/progress.md`.
